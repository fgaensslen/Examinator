---
question: "As a developer, how can you identify a composite action on GitHub?"
documentation: "https://docs.github.com/en/actions/creating-actions/creating-a-composite-action"
---
- [ ] The action’s repository name includes the keyword “composite.”
- [ ] The action’s repository includes an init.sh file in the root directory.
- [ ] The action’s repository includes Dockerfile and package.json files.
- [x] The action.yml metadata file has the runs.using value set to composite.
